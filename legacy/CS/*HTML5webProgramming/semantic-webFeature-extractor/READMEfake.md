# 1. Problem definition
> 기술적으로 무엇이 문제였는가
> (성능, 구조, 확장성, 복잡도 등)
- 방송대에서 들은 CS과목의 fundamental한 이론을 토대로 내가 코드/구현 부분에서 궁금했고 하고 싶었던 프로젝트를 해보자는게 출발점.
- 이 과목에서 배운건 결국 웹 문서의 전체적인 구조를 훑는 방법 정도인데,
- 상당히 지엽적으로 복잡하다고 느꼈음. 태그나 속성들의 계층 구조에서 레벨의 수에 비해 한 레벨의 하위레벨의 수가 많기 때문. 
- 이 구조는 기존 컴퓨팅 방식에서 탐색 알고리즘에 효율적인 자료구조가 대부분 트리 구조에 기반하기 때문이라고 추측함.
- 하지만 이번 학습에서 느꼈듯, 인간의 인지체계에서는 인지부하가 많이 걸리는 구조이고, 비슷한 신경망 구조의 모델인 딥러닝 기반의 AI모델들 또한 텍스트 형태 기반의 웹문서 데이터(html, css, js)를 학습 시 비슷한 난항을 겪지 않을까? 하는 생각에서 출발.
- 기존 크롤러는 시맨틱 태그보다 구조적 태그들에 포커스를 맞추어 파싱 및 결과를 반환함. 게다가 적용된 css의 비정형적 의미를 반영하기 어려운 구조임.
