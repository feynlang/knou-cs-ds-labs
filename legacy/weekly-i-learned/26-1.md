    - 요약
        - AI 사진 추천
            1. 얼굴 탐지 파이프라인: 식별 후 좌표(1.bounding box, 2.landmark)
            2. 얼굴 인식 파이프라인: (1)임베딩 벡터 (2)유사도 구하기 (3)threshold 판별
        - 최적화
            1. AS-IS 병목지점 측정(우선순위 판단 <- 실제 실행t,메모리사용량)
                - (1) inline timing snippet (간편; 오버헤드o)
                - (2) profiler-based approach (cpu/gpu사용량, 커널레벨 metric 위주 파악)
                - 결과: 세부 단계별 수행시간(**Load**->**Detection**->**recognition**->compare)
            2. 최적화 step
                - step1. 필터링 우선적용 & 연산융합
                    - Detection 작업 = <u>Detect Face</u> -> Crop bitmap
                    - Detect Face 작업 = Preprocessing -> Inference -> <u>Postprocessing</u>
                    - Postprocessing 작업 = **<u>Decode Bboxes&Landmarks</u>** -> Confidence처리&필터링 -> NMS 적용 -> 최종후처리
                        - Decoding = (1)신뢰도(confidence), (2)Location(영역좌표), (3)Landmark(특징점) 추출
                        - 후에 Confidence 기준 필터링 -> 걸러지는 데이터 다수.\
                        => 필터링을 먼저!(먼저 버리고 이후의 Bbox,Landmark 디코딩 작업)\
                        => 연산융합(연산 내부에서 독립적 작업단위-ex. Bbox에 대한 것과 Landmark에 대한 것-간 병렬 수행 가능)
                - step2. top-K 선택 알고리즘 최적화
                    - step1 후의 병목지점=>top-k선택 시 cpu사용량 높음
                    - 기존 코드= 불필요한 전체정렬 + 메소드체이닝 시 중간컬렉션 생성(오버헤드)\
                    => 최소 힙 자료구조 사용(이를 거친 상위 K개만 정렬작업)
                - step3. TensorBuffer사용 시 getFloatValue(x) -> floatArray(o)
                - step4. TensorFlow Lite 스레드풀 크기최적화
                - step5. 모델 인스턴스 병렬화 (처리량최적화)
